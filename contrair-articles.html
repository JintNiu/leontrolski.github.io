Below are a collection of interesting articles/posts that counter certain premises.

Scaling

From the word go, we need to be able to build our system to be horizontally scalable, that will make it quicker.
  • Stack Overflow basically runs on one big MSSQL server.
  • Dropbox started very simple.
  • Stuff from the 70s is often quick.
  • A single node kdb instance will outperform most clusters.

Async

If we make the service async, that will make it quicker and easier to work on.
  • Your webserver is already non-blocking, that will probably do unless you have arbitrarily long connections.
